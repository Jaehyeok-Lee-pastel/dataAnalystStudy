프로젝트 발표: "영화는 왜 성공할까? 캐글 영화 데이터로 보는 평점과 흥행의 관계"
1. 프로젝트 개요
캐글 영화 데이터를 활용한 분석 프로젝트

목표: 영화의 평점과 수익 간의 관계를 탐색하고, 간단한 예측 모델 실험

"흥행 성공 영화에는 어떤 특성이 있을까?"라는 호기심에서 시작

2. 데이터셋 소개
Kaggle: The Movies Dataset (TMDB 기준)

주요 컬럼:

vote_average: 평점

revenue, budget: 수익, 제작비

genres, release_date, runtime 등

총 약 4,800개 영화 데이터

3. 분석 방향 설정
‘영화의 성공’을 어떻게 정의할까?

수익이 제작비보다 많으면 ‘성공’

ROI(Return on Investment) > 1

분류 문제로 단순화:

성공(1) / 실패(0)

4. 데이터 전처리 개요
결측치 처리:

revenue, budget에 0이 많음 → 제거 또는 대체

날짜 형식 변환

텍스트/JSON 형식 컬럼 정리 (예: genres)

5. EDA 1 – 평점/수익 분포
평균 평점: 약 6.1

수익 분포: 극단적으로 치우침 → 로그 변환 고려

시각화: 히스토그램, 박스플롯 등

6. EDA 2 – 평점 vs 수익
산점도: 평점이 높다고 무조건 수익이 높은 것은 아님

상관계수: 약 0.2 수준의 약한 양의 상관관계

7. EDA 3 – 장르별 분석
장르별 평균 평점 & 평균 수익

예:

드라마/다큐멘터리는 평점이 높음

액션/어드벤처는 수익이 높음

8. 타겟 변수 정의
ROI 계산: revenue / budget

ROI > 1이면 ‘성공’으로 간주 → 타겟 컬럼 생성

클래스 비율 확인: 성공 약 35%, 실패 약 65%

9. 변수 선택 및 가공
수치형 변수: budget, runtime, vote_average

범주형 변수 인코딩:

genres: 가장 대표 장르만 추출 (예: 첫 번째 항목만 사용)

One-hot encoding

10. 모델 선정 및 설명
간단한 분류 모델 2종:

로지스틱 회귀 (해석 용이)

랜덤포레스트 (비선형 패턴 포착 가능)

복잡한 딥러닝 모델은 사용하지 않음 (신입 수준에 맞게)

11. 모델 학습 및 평가 방법
데이터 분할:

Train/Test = 80/20

평가 지표:

정확도(Accuracy), 정밀도(Precision), 재현율(Recall)

교차 검증은 생략 or 간단히 언급

12. 로지스틱 회귀 결과
정확도: 약 68%

중요한 변수:

vote_average, budget가 유의미한 영향

모델 계수 시각화 (bar chart)

13. 랜덤포레스트 결과
정확도: 약 72%

중요 변수 확인:

vote_average, budget, runtime

overfitting 우려로 depth 제한 설정

14. 중요 변수 시각화
변수 중요도 bar plot

분석 결과 요약:

평점은 성공 예측에 도움 되지만 절대적이진 않음

제작비도 중요한 변수

15. 시각화 정리
주요 시각화 한 장에 모음

평점 vs 수익 산점도

ROI 분포

장르별 분석

모델 중요도 그래프

16. 프로젝트에서 배운 점
데이터 전처리의 중요성 체감

간단한 모델로도 흥미로운 인사이트 도출 가능

실무에서는 다양한 외부 요인 반영 필요

17. 한계점
수익 데이터가 0인 영화 다수 존재 → 분석 신뢰도 낮아질 수 있음

외부 변수(홍보, 배우, 시즌 등) 미반영

평점은 개봉 후 생기는 데이터 → 실제 예측에는 한계

18. 추가 아이디어
리뷰 텍스트 감성 분석 활용 가능

감독/배우의 이전 흥행 기록 활용

개봉 시기(성수기/비수기) 변수화 가능

19. 결론
평점은 성공 가능성의 유의미한 지표 중 하나

그러나 예측 정확도는 제한적 → 평점만으로 성공을 설명하기는 어려움

복합적인 요인 고려가 필요
